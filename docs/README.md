# docs/

Overview of all spoke directories grouped by surface.

**Technical architecture** (how the system is built):
- `technical-architecture/` — C4 diagrams, deployment topology, tech stack detail

**Domain architecture** (what the system means):
- `domain-architecture/` — context map, cross-context events, L2 ubiquitous language, event contracts

**Cross-cutting:**
- `developer-workflows/` — build, test, deploy, CI/CD
- `user-experience/` — UI conventions, journeys, accessibility
- `adr/` — numbered decision records
- `plans/` — implementation plans and specs
- `musings/` — pre-artifact thought capture
- `retros/` — post-session reflection docs
- `tech-debt/` — pre-existing LSP errors and tech debt
- `agents-detail/` — project-specific agent spoke guidance

See `AGENTS.md` and `PURPOSE.md` for project context.
